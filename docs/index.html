The bite

Four things nobody else pairs.

The differentiators up top — the rest of a real dev team, automated, below them.

Human-gated merge

QA passes, a live preview of the running app spins up, you approve. Agents never push — the orchestrator alone merges.

Agents read less. Runs cost less.

On-device RAG — embeddings, retrieval, a shared context cache and a cached project brief — so agents fetch the exact code they need instead of burning tokens re-reading your repo. No source ever uploaded.

Agents that learn

The more they ship, the sharper they get. Learnings + a pinned context cache feed the next task — the swarm stops repeating mistakes.

Worktree isolation

A dedicated git worktree per agent, per task — zero collisions, clean diffs, safe parallelism.

A real browser, not just unit tests — QA drives the running app, screenshots it, fails on any console/network error.

Blast-radius planning

Scale across machines

Run the swarm on many boxes. Atomic task claim, a DB merge-lock, worker heartbeats — and a dead machine’s tasks get reclaimed automatically.

How it works

Task in, merge out — in four moves.

01

Task

Type it, or paste a message and let intake split it into scenarios.

02

Swarm

Agents fan out into isolated worktrees — plan, build, QA in parallel.

03

Review

QA passes → the task parks in review. You preview the branch.

04

Merge

You click approve. The orchestrator merges — the only thing that can.
